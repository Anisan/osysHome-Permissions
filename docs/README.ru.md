# Модуль «Права доступа»

Плагин **Permissions** управляет доступом к HTTP-маршрутам в osysHome: показывает ограничения из кода, позволяет задавать дополнительные правила для пользователей и ролей и сбрасывать их.

| | |
|---|---|
| **Админка** | [/admin/Permissions](/admin/Permissions) |
| **Категория** | System |
| **Версия** | 0.1 |

---

## Обзор

У каждого маршрута (endpoint) есть два уровня:

1. **Доступ по умолчанию** — роли, заданные в коде обработчика (`required_roles`). Отображается синим бейджем. Если в коде ограничений нет — **Без ограничений**.
2. **Дополнительные правила** — необязательные переопределения в объекте `_permissions`. Можно разрешить или запретить конкретных пользователей и роли для каждого HTTP-метода (GET, POST, PUT, DELETE).

Дополнительные правила работают поверх доступа по умолчанию. Один и тот же пользователь или роль не может одновременно быть в списке разрешения и запрета для одного метода.

## Интерфейс админки

### Панель инструментов

| Элемент | Назначение |
|---------|------------|
| **Поиск** | Модули по имени/заголовку, endpoint и URL |
| **Только с правилами** | Показать только модули/маршруты с доп. правилами |
| **Развернуть / Свернуть все** | Открыть или закрыть все карточки модулей |
| **Статистика** | Число модулей, маршрутов и записей с правилами |

### Карточка модуля

- Клик по заголовку — развернуть/свернуть список маршрутов.
- **Настроить модуль** — правила на уровне всего blueprint (модуля).
- **Сброс** (↺) — удалить доп. правила модуля (кнопка видна, если правила есть).

В строке маршрута для каждого метода показаны **Доступ по умолчанию** и, при наличии, дополнительные строки разрешения/запрета.

### Окно редактирования

Для каждого HTTP-метода:

| Поле | Смысл |
|------|--------|
| **Доступ пользователям** | Только эти пользователи (в дополнение к логике по умолчанию) |
| **Запрет пользователям** | Эти пользователи заблокированы |
| **Доступ ролям** | Только эти роли |
| **Запрет ролям** | Эти роли заблокированы |

**Все** (`*`) — все пользователи. **Любая роль** (`*`) — любая роль.

**Сохранить** — записать правила. **Сбросить правила** — удалить все доп. правила для текущего модуля или endpoint.

## Формат хранения

Правила хранятся в свойствах системного объекта `_permissions`:

| Ключ | Область |
|------|---------|
| `_permissions.blueprint:ИмяМодуля` | Весь модуль (blueprint) |
| `_permissions.ИмяМодуля:endpoint` | Один endpoint |

Значение — JSON по HTTP-методам:

```json
{
  "get": {
    "access_users": ["operator"],
    "denied_users": ["test"],
    "access_roles": ["admin"],
    "denied_roles": ["guest"]
  }
}
```

Пустой объект `{}` — без доп. правил (действует только доступ по умолчанию).

## API

### `GET /Permissions/list`

Все маршруты, сгруппированные по модулю. Требуется роль администратора.

В каждой группе: `title`, `category`, `permissions` (правила модуля), `endpoints[]` с `methods[]`, `required_roles`, `permissions`, `url`.

### Сохранение через API свойств

```
POST /api/property/_permissions.{key}
Content-Type: application/json

{
  "data": { "get": { "access_users": ["user1"] } },
  "source": "Permissions"
}
```

В ключе endpoint точка заменяется на двоеточие, например `MCPServer:mcp_endpoint`.

|------|-----|

| Администратор | `admin` |
