# Access permissions module

![Permissions Icon](static/Permissions.png)

See full documentation in [docs/README.md](docs/README.md).

The **Permissions** plugin manages HTTP route access in osysHome: default restrictions from code, additional per-user and per-role rules, and reset.

| | |
|---|---|
| **Admin UI** | [/admin/Permissions](/admin/Permissions) |
| **Category** | System |
| **Version** | 0.1 |

## Quick start

1. Open **Admin → Access permissions** (`/admin/Permissions`).
2. Expand a module card to see routes and **Default access** for each HTTP method.
3. Use **Edit** or **Configure module** to add additional allow/deny rules.
4. Use **Reset rules** (↺) to remove custom rules and restore default access only.
